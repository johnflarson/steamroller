Read scripts/main.gd first, then add auto-reroll logic.

**_check_and_handle_no_moves():** New function. Called from _highlight_valid_cells() when zero valid cells found.

Per CONTEXT.md locked decisions:
- Unlimited consecutive auto-rerolls until a valid move is found
- Each reroll logged
- Per RESEARCH.md Pitfall 4: MUST check for board-full stalemate before rerolling to avoid infinite loop

Implementation:
1. First check if ANY unclaimed cells exist on the entire board:
   ```gdscript
   func _has_unclaimed_cells() -> bool:
       for r in rows:
           for c in cols:
               if owner_grid[r][c] == -1:
                   return true
       return false
   ```
2. If no unclaimed cells exist, trigger stalemate resolution (per CONTEXT.md):
   - `state = GameState.GAME_OVER`
   - Call `_resolve_stalemate()` (implement this function)
   - Return
3. If unclaimed cells exist but none match current_roll:
   - Log: `_log("No valid moves for %d — auto-rerolling..." % current_roll)`
   - Generate new roll: `current_roll = randi_range(1, dice_faces)`
   - Log: `_log("%s rolled a %d" % [players[current_player].name, current_roll])`
   - Update UI: `_update_ui()`
   - Re-check: call `_highlight_valid_cells()` again (recursive — will either find valid cells or reroll again)

**CRITICAL: The recursion is safe because:**
- The stalemate check at step 2 guarantees unclaimed cells exist
- Each unclaimed cell has a value 1-6, so eventually a matching roll will occur
- Expected rerolls before match is small (typically 1-3)

However, to be extra safe, add a max reroll counter (e.g., 100) with a fallback log message and state = GAME_OVER if exceeded (should never happen with a d6 and unclaimed cells, but prevents editor hang in any edge case).

**_resolve_stalemate():** New function (per CONTEXT.md locked decisions).
1. Find player(s) with highest score
2. If single winner: `_log("All cells claimed! %s wins with %d points!" % [winner.name, winner.score])`
3. If tie: `_log("All cells claimed! Shared victory: %s with %d points!" % [tied_names_joined, score])`
4. `_disable_all_cells()` — new helper that disables all buttons and the Roll button

**_disable_all_cells():** New function.
- Iterate all cell_buttons, set disabled = true
- Set roll_button.disabled = true

Update `_highlight_valid_cells()` to integrate with auto-reroll:
- After iterating all cells, if valid_count == 0, call `_check_and_handle_no_moves()`
- If valid_count > 0, update RollResultLabel and log normally
