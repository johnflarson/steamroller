**Add `_animate_score_cells(cells: Array)` function:**
- For each `cell_vec` (Vector2i) in `cells`:
  - Get button ref: `cell_buttons[cell_vec.y][cell_vec.x]`
  - Set `btn.pivot_offset = btn.size / 2.0` (MUST be at animation time, not _ready — Pitfall 1)
  - Reset `btn.scale = Vector2.ONE` defensively (Pitfall 2)
  - Create tween: `var tw := create_tween()`
  - Pop up: `tw.tween_property(btn, "scale", Vector2(1.2, 1.2), 0.15).set_trans(Tween.TRANS_BACK).set_ease(Tween.EASE_OUT)`
  - Return: `tw.tween_property(btn, "scale", Vector2(1.0, 1.0), 0.15).set_trans(Tween.TRANS_SINE).set_ease(Tween.EASE_IN_OUT)`
- Do NOT await the tween. Score is already awarded before this call.

**Add `_apply_spent_appearance(cells: Array)` function:**
- For each `cell_vec` in `cells`:
  - Get `player_idx` from `owner_grid[cell_vec.y][cell_vec.x]`
  - Create spent color: `Color(PLAYER_COLORS[player_idx].r, PLAYER_COLORS[player_idx].g, PLAYER_COLORS[player_idx].b, SPENT_ALPHA)`
  - Call `_set_cell_color(btn, spent_color)` to apply dimmed appearance

**Modify `_check_score()`:**
- After `scored_grid` marking and score increment, BEFORE the return:
  - Call `_apply_spent_appearance(cells)` — apply dimmed look immediately
  - Call `_animate_score_cells(cells)` — tween plays over the already-dimmed button
  - Keep the existing `_log` call (which should now be `_log_score` from Plan 01)
  - Do NOT await. Return true immediately after.
