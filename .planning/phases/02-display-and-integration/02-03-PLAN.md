  <action>
**HTML5 export setup:**
1. Check if Godot export templates are installed: look for web export template in `~/.local/share/godot/export_templates/` (Linux) or equivalent.
2. If templates are not installed, note this for the checkpoint — the user will need to download them via Godot Editor > Export > Manage Export Templates.
3. Configure export preset for Web (HTML5) in project.godot:
   - Add a Web export preset
   - Disable thread support (single-threaded export) — this game has no threading needs and avoids the SharedArrayBuffer CORS header requirement (Pitfall 3 from research)
   - Set output path to `export/web/index.html`
4. Attempt to export via Godot CLI: `godot --headless --export-release "Web" export/web/index.html`
   - If this fails due to missing templates, document the error for the checkpoint.

**For the local test server:**
- Download Godot's `serve.py` from `https://raw.githubusercontent.com/godotengine/godot/master/platform/web/serve.py` to the project root (or a `tools/` directory).
- Alternatively, if single-threaded export is used, a simple `python3 -m http.server` from the export directory may suffice (no special CORS headers needed).
- Start the server pointing to the export directory.
- Note the URL (http://localhost:8000) for the verification checkpoint.

**If Godot CLI is not available on PATH:** Note this for the checkpoint. The user will need to export from the Godot Editor GUI (Project > Export > Web > Export Project).
