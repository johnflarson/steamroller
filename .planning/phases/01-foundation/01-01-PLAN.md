Create the main game script implementing the core data structures, state machine, board generation, and roll logic.

**Data Model (per CONTEXT.md locked decisions):**
- `var rows := 10` and `var cols := 10` (configurable, not hardcoded)
- `var dice_faces := 6` (configurable for future extensibility)
- `const WIN_SCORE := 5`
- `var board_numbers: Array = []` — [row][col] -> int 1-6
- `var owner_grid: Array = []` — [row][col] -> int (-1 unclaimed, 0-3 player index)
- `var scored_grid: Array = []` — [row][col] -> bool (spent/scored flag per CONTEXT.md)
- `var cell_buttons: Array = []` — [row][col] -> Button reference
- `var players: Array` — array of dicts with name, color, score. Default 4 players: Player 1 (Red), Player 2 (Blue), Player 3 (Green), Player 4 (Yellow)
- `var player_count := 4` (default 4 for testing per CONTEXT.md)
- `var current_player := 0`
- `var current_roll := 0`

**State Machine:**
- `enum GameState { WAIT_ROLL, WAIT_PICK, GAME_OVER }`
- `var state: GameState = GameState.WAIT_ROLL`

**@onready node references** for all UI nodes in the scene: GridContainer, CurrentPlayerLabel, RollResultLabel, RollButton, ScoresLabel, GameLog.

**_ready() function:**
1. Call `_init_arrays()` to set up 2D arrays
2. Call `_generate_board()` to fill board_numbers using shuffle bag algorithm (per RESEARCH.md Pattern 5): create pool with base_count per face + distribute remainder, shuffle, assign to grid
3. Call `_build_grid()` to create 100 Button nodes in GridContainer (per RESEARCH.md Pattern 2): each button gets text=str(board_numbers[row][col]), custom_minimum_size=Vector2(50,50), pressed signal connected via `.pressed.connect(_on_cell_pressed.bind(row, col))`, stored in cell_buttons[row][col]
4. Call `_update_ui()` to set initial label text

**_on_roll_button_pressed():**
1. Guard: `if state != GameState.WAIT_ROLL: return`
2. Generate roll: `current_roll = randi_range(1, dice_faces)`
3. Set state to WAIT_PICK
4. Call `_highlight_valid_cells()` (stub for now — just log the roll)
5. Call `_log()` with roll message
6. Call `_update_ui()`

**_on_cell_pressed(row, col):** Stub — just `pass` for now (Plan 02 implements)

**_highlight_valid_cells():** Stub that iterates grid and does nothing yet (Plan 02 implements the visual highlighting)

**_clear_highlights():** Stub (Plan 02 implements)

**_update_ui():**
- Set CurrentPlayerLabel text to current player name
- Set RollResultLabel text to current roll (or "-" if 0)
- Set RollButton disabled based on state (disabled when state != WAIT_ROLL)
- Set ScoresLabel text to all player scores formatted as multiline

**_log(message: String):**
- Use `game_log.append_text(message + "\n")` (per RESEARCH.md — append_text is correct for Godot 4, and the RichTextLabel is scene-created to avoid GitHub issue #94630)

**_set_cell_color(btn: Button, color: Color):** Utility per RESEARCH.md Pattern 3 — create StyleBoxFlat, set bg_color, override all 4 states (normal, hover, pressed, disabled).

Connect the RollButton pressed signal in _ready(): `roll_button.pressed.connect(_on_roll_button_pressed)`

Do NOT call `randomize()` — auto-seeded in Godot 4 per RESEARCH.md.
